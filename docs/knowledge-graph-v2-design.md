# 知识图谱 V2 设计

## 1. 目标

在知识图谱 MVP 的“可同步、可问答”基础上，补齐原生可视化探索能力，让用户可以：

- 直接在界面查看图谱结构
- 通过社区、节点类型和关键词过滤图谱
- 点击节点查看邻居、关系和相关文章
- 从统计视图自然进入图谱探索，而不是依赖额外工具

## 2. V2 范围

本轮 V2 包含：

- 图谱快照视图接口
- 前端 SVG 图谱画布
- 节点详情侧栏
- 节点类型 / 社区 / 关键词过滤
- V2 设计与实现文档

本轮 V2 不包含：

- 复杂力导布局仿真
- 画布编辑、拖拽建边
- 图谱写回
- 独立图库数据库

## 3. 后端设计

### 3.1 新增快照视图接口

新增接口：

- `GET /knowledge-graph/snapshot`

支持参数：

- `community_id`
- `node_type`
- `q`
- `limit_nodes`

### 3.2 返回内容

接口返回：

- `nodes`
- `links`
- `communities`
- `build`
- `generated_at`
- `available_node_types`

### 3.3 过滤策略

- 优先按社区、节点类型和关键词过滤节点
- 节点不足时补充一跳邻居，避免画布过于破碎
- 链接只保留选中节点集合内部的边

## 4. 前端设计

### 4.1 可视化组件

新增 `KnowledgeGraphExplorer`，以 SVG 方式渲染：

- 社区中心布局
- 节点圈层布局
- 关系边
- 高亮选中节点及其邻居

### 4.2 交互

- 点击节点：加载节点详情
- 节点详情展示：
  - 节点元信息
  - 邻居节点
  - 关系边
  - 相关文章
- 支持清空筛选与刷新

### 4.3 融入方式

将 V2 画布直接嵌入现有 `KnowledgeGraphPanel` 顶部，保证：

- 仍保留 MVP 的统计、同步、问答和路径能力
- 新增画布探索，不破坏原有工作流

## 5. 技术选择

### 5.1 不引入重型图库

当前选择原生 SVG，而不新增 D3 / Cytoscape / Sigma 这类依赖，原因是：

- 交付快
- 部署成本低
- 足够支撑当前的探索需求

### 5.2 后续可演进方向

如果 V2 使用效果稳定，后续可以再继续演进：

- 力导布局
- 缩放和平移
- 社区折叠
- 节点 pin / focus 模式
- 画布级别路径高亮
