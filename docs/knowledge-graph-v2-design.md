# 知识图谱 V2.1 设计

## 1. 目标

在知识图谱 MVP 的“可同步、可问答”和 V2 的“可视化探索”基础上，继续把图谱能力原生融入现有系统，让用户可以：

- 不离开当前工作流，就能从文章详情、AI 问答结果、路径结果和社区列表进入图谱
- 在同一个知识图谱工作台里完成查看、聚焦、钻取、问答和回看相关文章
- 以“图谱导航命令”的方式打通多个入口，而不是在每个页面重复实现一套状态机

## 2. V2.1 范围

本轮 V2.1 包含：

- 全局图谱导航上下文
- 图谱快照接口的聚焦子图能力
- 社区钻取抽屉
- 画布级路径高亮、邻域扩展、缩放和平移
- 文章详情与 AI 对话结果中的图谱跳转入口
- 前端交互测试基建与关键用例

本轮 V2.1 不包含：

- 复杂力导布局仿真
- 图谱节点编辑、边编辑和写回
- 独立图库数据库
- 与主路由完全分离的图谱子应用

## 3. 原生融入策略

### 3.1 设计原则

- 图谱不是独立页面能力，而是现有新闻工作流中的一个“深入理解层”
- 所有可以产出结构化图谱上下文的地方，都应该能发出统一的图谱导航命令
- Dashboard 负责切换到知识图谱页签，知识图谱页签负责解释并消费导航命令

### 3.2 全局命令模型

新增统一导航命令，覆盖以下信息：

- `reason`：触发原因，区分节点、社区、路径、文章和自定义命令
- `selectedNodeKey`：默认选中的节点
- `focusNodeKeys`：需要聚焦展示的节点集合
- `expand_depth`：局部图谱展开深度，支持 `0 / 1 / 2`
- `highlightedNodeKeys` 与 `highlightedEdgeKeys`：路径或结果高亮
- `communityId` 与 `openCommunityId`：社区过滤与社区抽屉联动

这样可以把“从哪里来”的差异收敛为“最终想看什么图”。

## 4. 后端设计

### 4.1 快照视图接口增强

继续使用：

- `GET /knowledge-graph/snapshot`

新增参数：

- `focus_node_keys`
- `expand_depth`

返回策略：

- 优先纳入显式指定的聚焦节点
- 当 `expand_depth > 0` 时，按 NetworkX 邻接关系做 1 跳或 2 跳扩展
- 在存在过滤条件时，继续补充排序靠前的候选节点，避免局部视图过于稀疏
- links 只保留当前选中节点集合内部的边

### 4.2 社区详情接口增强

继续使用：

- `GET /knowledge-graph/communities/{community_id}`

新增返回字段：

- `summary_text`
- `relation_types`

目的：

- 让社区抽屉可以直接给出“这个社区是什么”的自然语言摘要
- 让前端不再自己拼装社区关系概览

## 5. 前端设计

### 5.1 全局上下文

在应用根部新增 `KnowledgeGraphViewProvider`，职责是：

- 暴露 `focusNode`
- 暴露 `focusCommunity`
- 暴露 `focusPath`
- 暴露 `focusArticle`
- 管理当前图谱导航命令

`Dashboard` 监听命令后自动切到 `knowledge-graph` 页签，保证外部入口触发时用户总能看到正确目标。

### 5.2 知识图谱工作台

`KnowledgeGraphPanel` 保持现有工作台定位，但增强为命令消费中心：

- 路径查询成功后驱动画布路径高亮
- 社区点击可以同步打开社区抽屉
- 节点、文章、社区和问答结果都能进一步驱动画布
- 统计、同步、问答、路径与画布共存在同一页，不拆工作流

### 5.3 图谱画布

`KnowledgeGraphExplorer` 继续使用原生 SVG，不引入重型图库。新增能力：

- 局部聚焦子图
- 1 跳 / 2 跳邻域扩展
- 画布缩放、拖拽平移、重置视图
- 路径边高亮与节点高亮
- 节点详情内直接发起 Graph / Hybrid 问答

### 5.4 社区钻取

新增 `KnowledgeGraphCommunityDrawer`：

- 显示社区摘要、主导关系、代表节点和代表文章
- 提供 Graph / Hybrid 问答入口
- 提供社区聚焦、节点定位、文章定位动作

这样社区不再只是统计列表，而是一个可操作的中间层。

### 5.5 外部入口联动

文章详情：

- 当前文章可以直接在图谱中定位
- 关联实体、命中社区、相关文章都可直接跳转到图谱

AI 对话：

- 命中节点可直接聚焦到画布
- 命中社区可直接打开社区视图
- 参考文章可直接触发图谱定位

## 6. 关键交互流

### 6.1 从文章详情进入图谱

1. 用户在文章详情看到图谱上下文
2. 点击文章、实体或社区入口
3. 上下文发出图谱命令
4. Dashboard 自动切到知识图谱页签
5. 画布聚焦到对应节点或社区

### 6.2 从路径查询进入画布

1. 用户在知识图谱工作台查询两节点最短路径
2. 返回路径节点和路径边
3. 面板调用 `focusPath`
4. 画布展示局部子图并高亮路径

### 6.3 从 AI 问答进入图谱

1. 用户在 AI 对话中看到命中节点、命中社区和参考文章
2. 点击任一入口
3. 对话区不直接渲染图，而是发送统一命令
4. 知识图谱页签接管后续展示和钻取

## 7. 技术取舍

### 7.1 为什么继续使用原生 SVG

- 当前节点规模下足够支撑探索场景
- 控制成本低，容易与现有 Ant Design 组件混排
- 交互增强主要是状态与联动问题，不是图库能力短板

### 7.2 为什么采用全局命令上下文

- 避免文章详情、AI 对话、图谱面板之间相互硬编码依赖
- 保持入口多样，但目标行为统一
- 后续如果再新增“日报总结跳图谱”“告警卡片跳图谱”等入口，可直接复用

## 8. 后续演进方向

- 更稳定的自动布局策略
- 社区折叠与展开
- 更细粒度的路径过滤与边类型过滤
- 画布内直接切换 Graph / Hybrid 结果上下文
- 更完整的前端交互测试覆盖
